## RAID Log Rules

When creating RAID logs (Risks, Assumptions, Issues, Dependencies):

**Format requirements:**
- Use vertical card format (NOT wide tables)
- Do NOT use triple backticks
- Group by RAID category
- Maximum 5 items per category initially

**RAID Format:**

### RAID Log

#### Risks
**R1: [Risk Title]**
- **Likelihood:** High / Medium / Low
- **Impact:** High / Medium / Low
- **Mitigation:** Brief mitigation strategy
- **Owner:** Role or team responsible

**R2: [Risk Title]**
- **Likelihood:** Medium
- **Impact:** High
- **Mitigation:** Mitigation approach
- **Owner:** Responsible party

#### Assumptions
**A1: [Assumption Title]**
- **Description:** What is being assumed
- **Validation:** How/when this will be confirmed
- **Impact if Wrong:** Consequence if assumption fails

**A2: [Assumption Title]**
- **Description:** Assumption details
- **Validation:** Validation approach
- **Impact if Wrong:** Potential impact

#### Issues
**I1: [Issue Title]**
- **Status:** Open / In Progress / Resolved
- **Impact:** Description of current impact
- **Action:** Required action to resolve
- **Owner:** Responsible party
- **Due:** Target resolution date

#### Dependencies
**D1: [Dependency Title]**
- **Type:** Internal / External
- **Dependent On:** System, team, or deliverable
- **Required By:** Date or phase needed
- **Status:** On Track / At Risk / Blocked

**Rules:**
- Use consistent structure for each item
- Include owner for accountability
- Prioritise by impact/likelihood
- Keep descriptions concise (1-2 lines)
- Offer to expand specific categories if needed

**Do NOT use wide tables for RAID logs - the vertical format is more readable.**
