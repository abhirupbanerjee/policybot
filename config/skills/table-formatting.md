## Table Formatting Guidelines

When presenting comparisons or structured data:

**Use tables when:**
- Comparing 2+ items across 2-3 attributes
- Showing simple structured information
- Presenting short pros/cons comparisons

**Table format:**
| Header 1 | Header 2 | Header 3 |
|----------|----------|----------|
| Data     | Data     | Data     |

**Width Rules:**
- Maximum total table width: 70 characters
- Maximum per-cell content: 20 characters
- Limit to 3 columns maximum
- If content exceeds limits, use vertical format (see below)

**Rules:**
- Keep column headers short and clear
- Use concise cell content (1-3 words)
- Left-align text columns

**When tables become too wide, use VERTICAL FORMAT instead:**

#### Feature: User Management
- **Public Users:** N/A
- **MDA Staff:** N/A
- **Admin:** Create, edit, deactivate users

#### Feature: Ticket Status Check
- **Public Users:** Check status of own tickets
- **MDA Staff:** View tickets for assigned entity
- **Admin:** View all tickets across entities

**Avoid tables when:**
- Cells would contain more than 3-4 words
- Comparing more than 3 categories/columns
- Total columns Ã— average cell width > 70 characters
- Presenting feature matrices or role comparisons (use vertical format)
